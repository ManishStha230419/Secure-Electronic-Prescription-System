================================================================================
  CORRECT FOLDER STRUCTURE
================================================================================

Your extracted folder should look like this:

FINAL_PACKAGE/
├── App.py                          ← Main application file
├── requirements.txt                ← Python dependencies
├── README.txt                      ← Installation instructions
├── FOLDER_STRUCTURE.txt            ← This file
└── templates/                      ← HTML templates (IMPORTANT: correct name!)
    ├── base.html
    ├── index.html
    ├── login.html
    ├── register.html
    ├── dashboard.html
    ├── profile.html
    ├── edit_profile.html
    ├── change_password.html
    ├── create_prescription.html
    ├── view_prescription.html
    ├── prescriptions.html
    ├── notifications.html
    ├── settings.html
    ├── analytics.html
    ├── audit_log.html
    ├── help.html
    ├── about.html
    ├── errors/
    │   ├── 403.html
    │   ├── 404.html
    │   └── 500.html
    └── admin/
        ├── dashboard.html
        ├── process_logs.html
        ├── change_logs.html
        └── algorithm_usage.html


AFTER RUNNING "python App.py", these files will be created:

├── nepal_rx_process_logs.db        ← Database (created automatically)
├── password_save_process.log       ← Password save logs (created automatically)
├── prescription_encrypt_process.log  ← Prescription encrypt logs (created automatically)
├── prescription_decrypt_process.log  ← Prescription decrypt logs (created automatically)
└── complete_process_logs.log       ← Complete logs (created automatically)


IMPORTANT:
- The folder MUST be named "templates" (not "templates_process")
- Log files are created in the same folder as App.py
- Database is created automatically when you first run the app
